## 功能

- 全屏无边框显示，专注中窗口置顶，非专注时窗口不置顶
- 自动跟随系统应用暗黑/明亮模式
- 同屏显示数字时间、日期、模拟时钟和专注倒计时
- 可选择 25 / 45 / 60 / 90 分钟专注时长
- 支持自定义专注时长，默认上限 240 分钟，可在设置中调整到最高 1440 分钟
- 专注过程中会持续恢复全屏和置顶状态，并拦截应用内关闭/退出全屏操作
- 专注中会拦截 `Win+Tab`、`Win+Ctrl+D`、`Win+Ctrl+←/→` 和 `Win+Ctrl+F4`，避免通过任务视图或虚拟桌面绕过专注窗口
- 内置功能面板，可管理计划专注、设置、白名单、防重启和更新
- 支持每天执行的计划专注，到达计划时间后自动开始
- 支持重启后恢复未完成的专注时段
- 在应用内显示 `Whitelist.txt` 程序列表，可启动或调出白名单程序
- 允许 `Whitelist.txt` 中的程序临时显示在 FocusClock 上层
- 支持在面板中添加、批量导入和删除白名单程序
- 支持从 GitHub Releases 检查并下载更新
- 专注结束后允许退出

> 说明：Windows 桌面应用无法从系统层面阻止任务管理器、注销、关机或管理员级工具强制结束进程。本应用在自身窗口范围内拦截关闭，并在专注期间持续恢复全屏和置顶。

## 构建

推荐使用随项目提供的脚本：

```powershell
.\build.ps1
```

也可以手动构建：

```powershell
cmake -S . -B build -G "Visual Studio 17 2022" -A x64
cmake --build build --config Release
```

生成程序位于：

```text
build\FocusClock.exe
```

构建时会把项目根目录的 `Whitelist.txt` 复制到 `FocusClock.exe` 同目录。

## 使用

启动后窗口会进入全屏但不置顶。选择一个预设时长，或使用 `-5 / -1 / +1 / +5` 调整自定义时长，然后点击“开始专注”。

专注开始后，FocusClock 会进入置顶全屏并记录当前专注会话。倒计时结束后会自动恢复为非置顶全屏，发出提示音，并允许退出程序。

## 功能面板

未专注时按 `F6` 打开/关闭功能面板，也可以按 `Esc` 关闭面板。专注时段内不能打开面板。

功能面板包含以下页面：

- “概览”：查看当前默认专注时长和最大专注时长
- “计划”：添加或删除每天执行的计划专注任务
- “设置”：调整最大专注时长、默认专注时长，或恢复默认设置
- “白名单”：通过文件、文件夹或运行进程添加白名单程序，也可以删除已有条目
- “更新”：检查最新发布版本，并在发现可用版本时执行下载更新
- “防重启”：添加当前用户登录时运行的计划任务，用于重启后恢复专注

## 计划专注

在功能面板的“计划”页可以添加每天执行的专注时间段：

- 开始/结束小时按 1 小时调整，分钟按 5 分钟调整
- 计划时长不能超过当前“最大专注时长”
- 计划之间不能重叠
- 支持跨零点时间段，例如 `23:30 - 00:30`
- 到达计划时间后会自动开始专注
- 如果程序在计划时间段内启动，会按剩余时间继续本次计划专注
- 计划会保存到 `FocusClock.ini`

## 设置

在功能面板的“设置”页可以调整：

- “最大专注时长”：控制自定义时长和计划专注可使用的上限，范围为 1 到 1440 分钟
- “默认专注时长”：控制程序启动时和“恢复默认”后使用的时长

设置会保存到 `FocusClock.ini`。点击“恢复默认”会把最大专注时长恢复为 240 分钟，把默认专注时长恢复为 25 分钟。

## 白名单

`Whitelist.txt` 必须放在 `FocusClock.exe` 同目录。每行填写一个允许显示在 FocusClock 上层的程序名或完整路径：

```text
notepad.exe
C:\Windows\System32\notepad.exe
```

规则和行为：

- 支持 `#` 或 `;` 开头的注释行
- 支持用英文双引号包裹路径
- 修改 `Whitelist.txt` 后无需重启，程序会自动重新读取
- 只写程序名时依赖 Windows 能解析该程序；完整路径最可靠
- 完整路径通常也能显示更准确的程序图标
- 应用右侧会以图标网格显示白名单程序，点击后会优先调出已运行窗口，找不到窗口时再启动程序
- 专注中调出白名单程序时，FocusClock 会把白名单窗口也设为置顶，并把自身固定在该窗口下方
- 如果当前前台窗口来自白名单程序，FocusClock 也会临时让出置顶层级
- 如果目标程序以管理员权限运行，而 FocusClock 没有管理员权限，Windows 可能会限制调出或置顶操作

在功能面板的“白名单”页还可以：

- 选择单个 `.exe` 文件添加
- 选择文件夹，递归添加其中的 `.exe` 程序
- 从当前运行进程列表中选择程序添加
- 删除已有白名单条目

面板写入白名单时会保存为 UTF-8 格式的 `Whitelist.txt`。

## 白名单布局

- 按住 `Shift` 拖动白名单图标区域：移动白名单列表
- 按住 `Shift` 在白名单区域滚轮：调整白名单图标大小
- `Shift + 方向键`：微调白名单列表位置
- `Shift + + / -`：微调白名单图标大小

白名单列表的位置和图标大小会保存到 `FocusClock.ini`。

## 防重启

在功能面板的“防重启”页点击“添加开机自启任务”，会创建一个 Windows 任务计划程序任务：

```text
FocusClock Rerun
```

这个任务会在当前用户登录时用 `-rerun` 参数启动 FocusClock。它不是普通的常驻自启动：只有检测到未完成的专注会话，或当前时间正处于计划专注时间段内时，才会恢复或开始专注；否则会自动退出。

专注会话信息会临时保存到 `FocusClock.ini` 的 `FocusSession` 配置段，专注正常结束后会自动清理。

## 更新

在功能面板的“更新”页会联网获取 GitHub Releases 信息。发现可用版本时，“立即更新”按钮会启用。

更新过程会：

- 从 GitHub 或可用代理源下载发布资产
- 优先使用名为 `FocusClock.exe` 的发布文件
- 使用系统 `curl.exe` 下载文件
- 使用 PowerShell 在程序退出后替换当前可执行文件
- 临时生成 `FocusClock.old.exe` 作为备份，并在下次启动时尝试清理

如果网络不可用、GitHub 无法访问，或系统缺少 `curl.exe` / PowerShell，更新可能失败。

## 快捷键

- `Esc`：未专注时退出；功能面板打开时关闭面板
- `Alt+F4`：仅在未专注时退出
- `F6`：仅在未专注时打开/关闭功能面板
- `Shift + 拖动白名单区域`：移动白名单列表
- `Shift + 鼠标滚轮`：在白名单区域调整图标大小
- `Shift + 方向键`：微调白名单列表位置
- `Shift + + / -`：微调白名单图标大小

## 配置文件

`FocusClock.ini` 会在 `FocusClock.exe` 同目录自动生成，用于保存：

- `Focus`：最大专注时长、默认专注时长
- `Schedule`：计划专注任务
- `Whitelist`：白名单图标位置和大小
- `FocusSession`：正在进行的专注会话，用于防重启恢复；专注结束后自动清理
